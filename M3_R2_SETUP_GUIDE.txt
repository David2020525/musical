================================================
CLOUDFLARE R2 STORAGE SETUP GUIDE
================================================

Follow these steps to set up R2 storage for MusicHub track uploads:

STEP 1: Create R2 Bucket
------------------------
1. Go to: https://dash.cloudflare.com
2. Navigate to: R2 Object Storage → Create bucket
3. Bucket name: musichub-tracks
4. Location: Automatic (or select nearest to Turkey/Europe)
5. Click "Create bucket"

STEP 2: Configure Public Access (IMPORTANT)
-------------------------------------------
1. In your bucket settings, go to "Settings" tab
2. Under "Public access", click "Allow Access"
3. Enable "Public bucket" option
4. Note the public bucket URL (format: https://pub-xxxxx.r2.dev)

STEP 3: Create API Token
-------------------------
1. Go to: R2 → Manage R2 API Tokens
2. Click "Create API Token"
3. Token name: musichub-uploads
4. Permissions: 
   - Object Read & Write
   - Apply to specific bucket: musichub-tracks
5. TTL: Forever (or set expiration as needed)
6. Click "Create API Token"

STEP 4: Copy Credentials
-------------------------
After creating the token, copy these values:

✅ Access Key ID: (starts with something like abc123...)
✅ Secret Access Key: (long string)
✅ Account ID: (your Cloudflare account ID)
✅ Bucket Name: musichub-tracks
✅ Public URL: https://pub-xxxxx.r2.dev

STEP 5: Provide Credentials
----------------------------
Paste your credentials in this format:

--- CLOUDFLARE R2 ---
Account ID: [paste here]
Access Key ID: [paste here]
Secret Access Key: [paste here]
Bucket Name: musichub-tracks
Public URL: [paste here]

================================================

IMPORTANT NOTES:
- The public URL is needed for serving uploaded tracks
- Keep your Secret Access Key secure
- The bucket must be public for tracks to be playable
- You can always regenerate API tokens if needed
